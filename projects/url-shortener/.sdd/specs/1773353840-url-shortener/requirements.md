## 1. 사용자 기능

**1.1** 사용자가 긴 URL을 제출하면, 시스템은 충돌 없는 고유한 짧은 코드를 생성하여 반환해야 한다.
- AC1: POST /api/shorten에 유효한 URL 제출 시 8자 이하의 고유 코드와 단축 URL을 반환한다.
- AC2: 동일한 URL을 두 번 제출해도 매번 새로운 코드가 생성된다.
- AC3: 유효하지 않은 URL 제출 시 400 에러와 오류 메시지를 반환한다.

**1.2** 사용자가 짧은 코드로 접근하면, 시스템은 원본 URL로 리다이렉트해야 한다.
- AC1: GET /:code 요청 시 301 또는 302 응답으로 원본 URL로 리다이렉트한다.
- AC2: 존재하지 않는 코드 접근 시 404 응답을 반환한다.
- AC3: 리다이렉트 시 해당 코드의 클릭 수가 1 증가한다.

**1.3** 사용자가 URL을 단축할 때 선택적으로 태그를 지정할 수 있어야 한다.
- AC1: POST /api/shorten 요청에 tag 필드 포함 시 해당 태그가 저장된다.
- AC2: tag 필드가 없으면 null로 저장되며 오류가 발생하지 않는다.
- AC3: 동일한 태그를 가진 URL 목록을 조회할 수 있다.

**1.4** 사용자가 전체 단축 URL 목록과 각각의 총 클릭 수를 조회할 수 있어야 한다.
- AC1: GET /api/stats 요청 시 모든 단축 URL의 목록을 반환한다.
- AC2: 각 항목에 code, originalUrl, tag, totalClicks, createdAt 필드가 포함된다.
- AC3: 결과는 생성일 내림차순으로 정렬된다.

**1.5** 사용자가 특정 단축 코드의 상세 통계를 조회할 수 있어야 한다.
- AC1: GET /api/stats/:code 요청 시 해당 코드의 상세 정보를 반환한다.
- AC2: 응답에 날짜별 클릭 수 배열이 포함된다 (date, clicks 형식).
- AC3: 존재하지 않는 코드 조회 시 404를 반환한다.

---

## 2. 시스템 동작

**2.1** 시스템은 짧은 코드 생성 시 충돌이 없어야 한다.
- AC1: 1000개의 URL을 단축해도 모든 코드가 고유하다.
- AC2: 코드는 URL-safe 문자로만 구성된다.

**2.2** 클릭 수는 리다이렉트 발생 즉시 반영되어야 한다.
- AC1: 리다이렉트 후 /api/stats/:code 조회 시 totalClicks가 즉시 증가해 있다.

**2.3** 시스템은 서버 재시작 후에도 데이터를 보존해야 한다.
- AC1: 서버 재시작 후 기존 단축 URL과 통계 데이터가 유지된다.

---

## 3. 외부 인터페이스

**3.1** API는 JSON 형식으로 요청과 응답을 처리해야 한다.
- AC1: Content-Type: application/json 헤더로 요청 시 정상 처리된다.
- AC2: 모든 API 응답은 JSON 형식이다.

**3.2** API는 CORS를 지원하여 프론트엔드에서 접근 가능해야 한다.
- AC1: 프론트엔드 출처에서의 API 요청이 CORS 오류 없이 성공한다.

---

## 4. 제약 조건

**4.1** 짧은 코드는 URL-safe 문자로만 구성되어야 한다.
- AC1: 생성된 모든 코드는 영문자, 숫자, _, - 문자만 포함한다.

**4.2** URL 입력에 대한 유효성 검사가 이루어져야 한다.
- AC1: http:// 또는 https://로 시작하지 않는 URL 제출 시 400 에러를 반환한다.
- AC2: 빈 URL 제출 시 400 에러를 반환한다.
