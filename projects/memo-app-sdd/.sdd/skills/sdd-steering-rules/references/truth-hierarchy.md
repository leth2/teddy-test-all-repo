# 진실의 계층 상세 가이드

## 계층 구조 다이어그램

```
스티어링 (방향/원칙)       ← 가장 높은 권위
    ↓
  스펙 (요구사항 → 설계 → 태스크)
    ↓
  코드 (구현 결과물)        ← 결과물, 진실 아님
```

## 각 계층의 역할

| 계층 | 무엇인가 | 변경 주체 |
|------|----------|-----------|
| 스티어링 | 방향, 제약, 기술 결정 | 인간 (의식적 결정) |
| 스펙 | 요구사항, 설계, 태스크 | AI (스티어링 기반 생성) |
| 코드 | 구현 결과물 | AI (스펙 기반 구현) |

## 드리프트 처리 표

| 드리프트 상황 | 올바른 대응 | 잘못된 대응 |
|--------------|-------------|-------------|
| 코드 ≠ 스펙 | 코드를 스펙에 맞게 수정 | 스펙을 코드에 맞게 변경 |
| 스펙 ≠ 스티어링 | 스펙 검토 + 스티어링 의식적 갱신 | 자동 덮어쓰기 |
| 스티어링 변경 필요 | `--intent` 플래그 명시적 사용 | 코드 분석 후 자동 업데이트 |

## 각 모드 상세

### Bootstrap 모드
새 프로젝트이거나 스티어링이 없을 때.

단계:
1. `.sdd/steering/` 템플릿 로드 (`steering/product/`, `tech/`, `structure/`)
2. `sdd-steering-rules` 스킬 읽기
3. 코드베이스 분석 (Glob, Grep, Read)
4. 패턴 추출 → What-only 스티어링 생성

### Sync 모드
기존 스티어링이 있을 때. 드리프트 감지가 목적.

단계:
1. `.sdd/steering/` 전체 읽기
2. 현재 스펙 확인
3. 코드베이스 분석
4. 드리프트 보고 (자동 수정하지 않음)
5. 코드 ≠ 스펙이면: "코드를 스펙에 맞게 수정" 권고

### Rewrite 모드
`--intent "<새 방향>"` 사용. 의식적 방향 전환.

코드베이스에서 참고 안 하는 항목:
- 버릴 기술 스택, 아키텍처, 구조

코드베이스에서 유지하는 항목:
- 도메인 지식 (비즈니스 규칙, 용어)
- 검증된 외부 연동 정보

Rewrite 후: `/sdd:spec-reset --steering`으로 이전 스티어링 아카이브 권장

## 스티어링 업데이트 트리거

**업데이트해야 할 때:**
- 비즈니스 방향 자체가 바뀌었을 때
- 기술 스택을 전환하기로 결정했을 때
- 갈아엎기 전 새 방향 정의할 때

**업데이트하면 안 될 때:**
- 매 코드 변경마다
- 버그 수정
- 아키텍처 변경 없는 리팩토링
- AI가 코드를 분석하고 "이게 현실이니까"라고 판단할 때

## 좋은 vs 나쁜 스티어링 엔트리 예시

### tech.md 예시

**✅ 좋음:**
```markdown
## 기술 스택
- 언어: TypeScript 5.x
- 런타임: Node.js 20 LTS
- 프레임워크: Hono
- 데이터베이스: PostgreSQL 15
- 테스트: Vitest

## 주요 결정
- 모노레포: 불필요한 복잡도, 단일 레포 유지
- ORM 없음: 직접 SQL로 성능과 제어 우선
```

**❌ 나쁨:**
```typescript
// 데이터베이스 연결 방법
const db = new Client({
  host: process.env.DB_HOST,
  ...
})
await db.connect()
```

### structure.md 예시

**✅ 좋음:**
```markdown
## 폴더 구조
- feature-first: 기능별 폴더, 각 폴더에 독립 컴포넌트
- 공유 코드: src/shared/

## 네이밍 규칙
- 파일: kebab-case
- 컴포넌트: PascalCase
- 함수: camelCase, 동사로 시작
```

**❌ 나쁨:**
```
폴더 구조 만드는 방법:
1. src/ 아래 features/ 폴더 만들기
2. 각 기능마다 하위 폴더 만들기
3. index.ts 파일 생성...
```

## Lazy Loading 패턴

스티어링에서 스킬 파일 참조:
```
> 상세: read .sdd/skills/<topic>-detail.md
```
AI는 해당 스킬이 필요할 때만 읽음.
