## v1 대비 핵심 개선사항

### 1. Requirements 검증 (NEW)
- v2에서 요구사항 2.3 추가: ACP 핸드셰이크 시퀀스 명시
- v1은 핸드셰이크 요구사항 없음 → 설계/구현에서 누락 가능

### 2. 외부 스펙 참조 (KEY IMPROVEMENT)
- `agentclientprotocol.com` 공식 문서 조사
- `references/acp-protocol.md` 저장
- v1은 추정 메서드명 사용 (`newSession`, `message`)
- v2는 실제 스펙 기반 (`session/new`, `session/prompt`)

### 3. 설계 계약 개선
- `sendPrompt()` (v1: `sendMessage()`) — 스펙 명세 반영
- `respondPermission(requestId: number, optionId: string)` — options 배열 방식
- (v1: `sendPermissionResponse(approved: boolean, requestId: string)` — 불완전)

### 4. Notification vs Request 구분
- v2: `!hasId` = Notification, `hasId + hasMethod` = Request
- v2: `session/request_permission`은 id 있는 Request (응답 필요)
- v1: 이 구분이 설계에서 명시되지 않음

### 5. 태스크 상태 정확도
- v2: `[~]` = 코드 작성됨 테스트 미실행 (12개 태스크)
- v2: `[x]` = 없음 (npm install 환경 없어서 정직하게 [~] 사용)
- v1: `[x]` = 모두 완료 표시 (실제로는 테스트 실행 여부 불명확)
