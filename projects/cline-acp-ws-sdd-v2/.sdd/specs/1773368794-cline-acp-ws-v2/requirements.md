# Requirements: cline-acp-ws-v2

> 스티어링 정렬 검증: product/tech/structure 스티어링과 일치함 ✅

---

## 1. ACP 에이전트 프로세스 관리

**1.1** 시스템은 WebSocket 클라이언트가 연결될 때 ACP 에이전트 프로세스(claude-agent-acp)를 자동으로 생성해야 한다.
- AC1: WebSocket 연결 시 에이전트 프로세스가 시작된다
- AC2: 에이전트 시작 실패 시 클라이언트에 에러 메시지를 전송한다

**1.2** 시스템은 WebSocket 클라이언트가 연결을 끊을 때 연결된 ACP 에이전트 프로세스를 종료해야 한다.
- AC1: WebSocket 연결 해제 시 에이전트 프로세스가 정상 종료된다
- AC2: 에이전트가 응답하지 않으면 강제 종료(SIGKILL)한다

**1.3** 시스템은 동시에 하나의 WebSocket 연결만 허용해야 한다.
- AC1: 두 번째 클라이언트 연결 시 에러 메시지와 함께 거부한다
- AC2: 거부된 연결은 즉시 닫힌다

---

## 2. ACP ↔ WebSocket 브릿지

**2.1** 시스템은 클라이언트로부터 받은 메시지를 ACP JSON-RPC 형식으로 변환하여 에이전트 stdin에 전송해야 한다.
- AC1: 사용자 텍스트 메시지가 ACP 형식으로 에이전트에 전달된다
- AC2: 잘못된 형식의 메시지는 에러로 처리된다

**2.2** 시스템은 ACP 에이전트 stdout에서 받은 JSON-RPC 응답을 WebSocket 메시지로 클라이언트에 전송해야 한다.
- AC1: 에이전트 응답이 실시간으로 클라이언트에 전달된다
- AC2: 스트리밍 응답은 청크 단위로 전달된다

**2.3** 시스템은 ACP 에이전트 프로세스 시작 직후 ACP 핸드셰이크(initialize → newSession)를 수행해야 한다.
- AC1: 에이전트 프로세스 시작 후 initialize 요청을 전송한다
- AC2: initialize 응답 수신 후 newSession 요청을 전송한다
- AC3: 핸드셰이크 완료 후에만 사용자 메시지를 에이전트에 전달한다

**2.4** 시스템은 ACP 에이전트의 툴콜(tool call) 이벤트를 별도 메시지 타입으로 클라이언트에 전송해야 한다.
- AC1: 툴콜 시작/종료 이벤트가 클라이언트에 전달된다
- AC2: 툴콜 이름, 인자, 결과가 포함된다

---

## 3. Human-in-the-Loop 권한 승인

**3.1** 시스템은 파일 쓰기/삭제 툴콜 발생 시 클라이언트에 승인 요청 메시지를 전송해야 한다.
- AC1: permission_request 메시지 타입으로 파일 경로와 작업 유형이 전달된다
- AC2: 클라이언트 응답 전까지 에이전트 실행이 일시 중단된다

**3.2** 시스템은 클라이언트의 승인/거부 응답을 ACP 에이전트에 전달해야 한다.
- AC1: 승인 시 에이전트가 툴콜을 실행한다
- AC2: 거부 시 에이전트에 거부 응답이 전달되고 에이전트가 대안을 탐색한다

---

## 4. 채팅 UI

**4.1** UI는 사용자가 텍스트 메시지를 입력하고 전송할 수 있는 채팅 입력 영역을 제공해야 한다.
- AC1: Enter 키 또는 전송 버튼으로 메시지가 전송된다
- AC2: 전송 중에는 입력이 비활성화된다

**4.2** UI는 사용자 메시지와 에이전트 응답을 시간 순서대로 표시해야 한다.
- AC1: 사용자 메시지와 에이전트 메시지가 구분되어 표시된다
- AC2: 스트리밍 중인 응답이 실시간으로 업데이트된다

**4.3** UI는 툴콜 실행 로그를 별도 영역에 표시해야 한다.
- AC1: 툴콜 이름, 상태(진행중/완료/실패), 결과 요약이 표시된다

**4.4** UI는 파일 권한 승인 요청 시 모달 다이얼로그를 표시해야 한다.
- AC1: 파일 경로, 작업 유형, 승인/거부 버튼이 포함된다
- AC2: 다이얼로그가 열려있는 동안 채팅 입력이 비활성화된다

---

## 5. 시스템 동작

**5.1** WebSocket 서버는 포트 3001에서 실행되어야 한다.
- AC1: 서버 시작 시 3001 포트에서 연결을 수락한다

**5.2** 시스템은 Docker Compose로 에이전트 서버와 UI를 동시에 실행할 수 있어야 한다.
- AC1: `docker-compose up`으로 전체 시스템이 시작된다
- AC2: 서버는 3001, UI는 5173 포트로 접근 가능하다
