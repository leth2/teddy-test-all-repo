# 브리핑 보고서 형식

## 전체 문서 구조

```markdown
# 작업 브리핑 — YYYY-MM-DD HH:MM (로컬 타임존)

## 요약
- 분석 기간: 최근 N시간
- 활성 스펙: N개
- 완료된 태스크: N개
- 중단된 항목: N개

## ✅ 완료된 작업
### [스펙명]
- 완료된 단계: requirements ✅ design ✅ tasks ✅ impl 🔄
- 완료된 태스크:
  - [x] 1.1 사용자 모델 구현
  - [x] 1.2 DB 스키마 생성
  - [x] 2.1 로그인 API

## ⚠️ 중단된 작업
### [스펙명]
- 중단 위치: 태스크 N.N
- 중단 이유: (로그에서 추출한 오류 메시지)
- 재시작 방법: `/sdd:spec-impl <feature> N.N`

## 📋 현재 진행 중
### [스펙명]
- 현재 phase: tasks-generated
- 진행률: 3/8 태스크 (37%)
- 다음 단계: `/sdd:spec-impl <feature>`

## 🔜 권장 다음 액션
1. (최우선) `/sdd:spec-impl 1741834500-user-auth 3.1` — 중단된 태스크 재시작
2. `/sdd:spec-impl 1741834560-dashboard` — 설계 완료 후 구현 대기 중
3. `/sdd:steering` — 드리프트 감지 (마지막 확인 48h 이상)

## 📊 전체 현황
| 스펙 | 생성일 | Phase | 진행률 | 상태 |
|------|--------|-------|--------|------|
| 1741834500-user-auth | 2026-03-13 11:15 | impl | 6/8 (75%) | 🔄 진행 중 |
| 1741834560-dashboard | 2026-03-13 11:16 | tasks | 0/5 (0%) | ⏳ 대기 중 |
| 1741834621-notification | 2026-03-13 11:17 | init | - | 🆕 시작 전 |
```

## 상태 아이콘 의미

| 아이콘 | 의미 |
|--------|------|
| ✅ | 완전 완료 |
| 🔄 | 진행 중 |
| ⏳ | 대기 중 (다음 단계 실행 필요) |
| 🆕 | 시작 전 |
| ❌ | 실패 (수동 개입 필요) |
| 🗄️ | 아카이브됨 |

## 진행률 계산

```
진행률 = 완료된 태스크 수 / 전체 태스크 수
단계 진행률:
- init: 0%
- requirements-generated: 25%
- design-generated: 50%
- tasks-generated: 75%
- impl-started: 75% + (완료 태스크 / 전체 태스크 * 25%)
- impl-complete: 100%
```

## 다음 액션 우선순위 결정

1. **오류로 중단된 작업** (즉시 주의 필요)
2. **impl 단계 진행 중** (계속 진행)
3. **태스크 생성 완료, impl 안 시작** (시작 권장)
4. **설계 완료, 태스크 없음** (태스크 생성 권장)
5. **요구사항 완료, 설계 없음** (설계 생성 권장)
6. **init만 있음** (플랜 생성 권장)
