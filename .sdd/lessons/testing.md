# Lessons: Testing

AI가 반복하는 실수 패턴 모음.

## T01 없는 라우트를 호출하는 테스트

**증상:** `POST /start → 404` 테스트 실패

**원인:** 다른 버전(WS)의 흐름을 새 버전(SSE)에 그대로 복사. SSE 서버는 `/start` 없이 자동으로 bridge 시작.

**체크:** 테스트 실행 전 서버 라우트 목록 확인: `grep -r "app\.(get\|post)" src/`

**수정:** 테스트 스크립트를 실제 서버 라우트에 맞게 수정. 존재하지 않는 라우트 호출 제거.

## T02 TIMEOUT_MS 너무 짧음 (첫 실행)

**증상:** T2.1 ACP 핸드셰이크 타임아웃 (기본 10초). 패키지 첫 다운로드 시 60초+ 필요.

**원인:** `npx` 첫 실행 시 패키지 다운로드 포함. 기본 타임아웃 10초로 부족.

**체크:** `TIMEOUT_MS=10000`이 test-queue에 있으면 첫 실행에 실패 가능

**수정:** ACP 핸드셰이크 테스트는 `TIMEOUT_MS=60000` 이상으로 설정

## T03 SSE agent-ready 이벤트명 불일치

**증상:** `agent_ready` (언더스코어) 기다리는데 서버는 `agent-ready` (하이픈) 전송

**원인:** 이벤트명 컨벤션 불일치. 서버 코드와 테스트 코드가 다른 이름 사용.

**체크:** `grep -r "agent.ready" src/ test/` 로 이름 일치 여부 확인

**수정:** 테스트와 서버 이벤트명 통일 (서버 코드 기준으로 맞춤)
