# AI가 코드를 100만 줄 썼다 — 사람이 한 줄도 안 건드리고

> 2026-03-13 | 원문: [하네스 엔지니어링: 에이전트 우선 세계에서 Codex 활용하기](https://news.hada.io/topic?id=27457)

---

## 무슨 일이 있었냐면

OpenAI 내부 팀이 5개월 동안 실험을 했어.

**조건:** 엔지니어는 코드를 직접 한 줄도 쓰지 않는다.  
**방법:** 모든 코드를 AI(Codex)가 쓴다.  
**결과:** 실제로 사용자가 쓰는 소프트웨어를 만들었다.

숫자로 보면:
- 코드 **100만 줄**
- **1,500개** Pull Request (코드 변경 단위)
- 엔지니어 **3명**
- 기간 **5개월**
- 엔지니어 1명이 하루 평균 **3.5개** PR 처리

일반적인 개발팀이라면 이 규모는 수십 명이 필요한 분량이야.

---

## 엔지니어가 뭘 했냐고?

코드를 안 썼다면 뭘 했을까? 이게 핵심이야.

**기존 엔지니어 역할:**
> "이 기능을 어떻게 코딩할까?"

**새 엔지니어 역할:**
> "AI가 이 일을 할 수 있는 환경을 어떻게 만들까?"

구체적으로:
- AI에게 **무엇을 만들어야 하는지** 명확하게 설명
- AI가 실수하면 → **왜 실수했는지** 분석 (AI 탓이 아니라 환경 탓)
- AI가 확인할 수 있는 **피드백 루프** 구성 (테스트, 로그, 화면 등)
- 문서와 구조를 **AI가 읽기 좋게** 정리

실패했을 때 "더 열심히 해!"가 아니라 "AI가 뭘 몰라서 틀렸지?"를 물었어.

---

## AI가 눈이 생겼다

흥미로운 부분이 있어. 팀이 AI에게 **화면을 보는 능력**을 줬어.

- 브라우저 화면 스크린샷 → AI가 직접 확인
- 로그(오류 기록) → AI가 직접 읽기
- 성능 수치 → AI가 직접 분석

그래서 이런 작업이 가능해졌어:
> "사용자가 버튼을 눌렀을 때 2초 안에 응답해야 해. 지금 얼마나 걸리는지 확인하고 고쳐."

AI가 화면을 보고 → 문제를 찾고 → 코드를 고치고 → 다시 확인하는 루프를 혼자 돌렸어.  
한 번에 **6시간 이상** 사람 없이 작업하기도 했어.

---

## 문서가 너무 많으면 오히려 망한다

팀이 초반에 저지른 실수가 있어. AGENTS.md라는 AI용 설명서를 아주 상세하게 만들었어.

결과? 오히려 더 나빠졌어.

**왜냐면:**
- 설명이 너무 많으면 AI가 뭐가 중요한지 모름
- 오래된 규칙이 쌓여서 무덤이 됨
- "모든 것이 중요"하면 사실상 "아무것도 중요하지 않은" 것과 같음

**해결책:** 설명서를 **목차**로만 만들었어.

```
AGENTS.md (약 100줄)
  → "자세한 내용은 docs/architecture.md 참고"
  → "설계 원칙은 docs/design.md 참고"
  → "코딩 규칙은 docs/quality.md 참고"
```

설명서 자체는 짧게 → 필요한 곳으로 안내 → AI가 필요할 때만 읽기.

---

## "AI가 모르면 존재하지 않는 것"

이게 이 실험에서 제일 중요한 통찰이야.

팀은 이렇게 말했어:

> "Slack 채팅에서 결정한 내용, 사람 머릿속에 있는 지식 —  
> AI가 읽을 수 없으면 그건 사실상 없는 거야."

그래서 모든 결정을 **파일로 남겼어**:
- 왜 이런 구조로 만들었는지
- 어떤 원칙으로 개발하는지
- 현재 어떤 문제가 있는지
- 앞으로 뭘 만들 계획인지

심지어 **팀 문화**(이모지 사용 방식까지!)도 문서화했어. 새 팀원에게 설명하듯이 AI에게 설명했더니 훨씬 나은 결과물이 나왔거든.

---

## 이게 왜 중요한가

**개발자가 아닌 사람에게:**  
앞으로 소프트웨어를 만드는 방식이 바뀌고 있어. "코딩을 잘 하는 사람"보다 "AI에게 무엇을 만들어야 하는지 명확히 설명하는 사람"이 중요해질 수 있어.

**주니어 개발자에게:**  
코딩 실력 자체보다, 문제를 잘 정의하고 AI 작업 환경을 잘 설계하는 능력이 점점 더 중요해지고 있어. 코드를 직접 쓰는 것 이상으로 "AI가 올바른 방향으로 일하게 만드는 것"이 핵심 역량이 돼가고 있어.

**모두에게:**  
소프트웨어 개발의 병목이 "코딩"에서 "방향 설정과 검증"으로 이동하고 있어. 뭘 만들지 명확히 알고, AI 결과물을 검증할 수 있는 사람이 강해지는 시대야.
